Requerimientos no funcionales

  • Seguridad

    • Los datos de los usuarios y propiedades deben ser almacenados de manera segura.
    Acceso seguro a la plataforma mediante autenticación.
    Protección contra posibles ataques como inyecciones SQL, cross-site scripting (XSS), etc.

  • Rendimiento

    • Las operaciones críticas (crear, editar, aceptar arriendos) deben completarse en un tiempo aceptable.
    Respuesta rápida en la búsqueda de propiedades y visualización de información.

  • Usabilidad

    • Interfaz de usuario intuitiva y fácil de usar.
    Soporte para diferentes dispositivos y tamaños de pantalla.
    Documentación clara y accesible para los usuarios.

  • Disponibilidad

    • El sistema debe estar disponible durante un alto porcentaje del tiempo (por ejemplo, 99.9% de disponibilidad).
    Implementación de medidas para garantizar la recuperación rápida en caso de fallas.

  • Escalabilidad

    • Capacidad para manejar un número creciente de usuarios y propiedades.
    Escalabilidad horizontal y vertical del sistema.

  • Mantenibilidad

    • Facilidad para realizar actualizaciones y correcciones.
    Registro de actividades y errores para facilitar la resolución de problemas.

  • Interoperabilidad

    • Capacidad para integrarse con otros sistemas si es necesario.
    Cumplir con estándares y protocolos relevantes.

  • Legal y Normativo

    • Cumplir con regulaciones y leyes locales relacionadas con la gestión de propiedades y datos personales.

  • Rendimiento

    • Establecer límites y criterios de rendimiento para operaciones críticas.
    Respuesta eficiente en la gestión de grandes cantidades de datos.

  • Compatibilidad

    • Compatibilidad con diferentes navegadores web y sistemas operativos.
