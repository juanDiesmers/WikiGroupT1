Proyecto: ArriendaTuFinca

Usuarios Principales

Usuario Arrendatario Usuario Arrendador
El “Usuario Arrendatario” tendrá la capacidad de realizar una variedad de acciones relacionadas con la gestión de propiedades y la interacción con otros usuarios. El “Usuario Arrendador” tendrá un conjunto diferente de funcionalidades diseñadas para facilitar el proceso de búsqueda y arrendamiento de propiedades.
  • Crear, editar, eliminar y actualizar propiedades disponibles para arrendamiento.
  • Revisar y responder a todas las solicitudes de arrendamiento recibidas.
  • Verificar los pagos realizados por los arrendadores.
  • Proporcionar calificaciones y comentarios sobre la experiencia de arrendamiento.
  • Buscar fincas disponibles filtradas por ubicación geográfica.
  • Solicitar el arrendamiento de una finca específica.
  • Realizar el pago correspondiente del arriendo.
  • Calificar tanto la finca arrendada como al arrendatario.

Implementación de la Aplicación

La implementación requerirá un sistema de autenticación y autorización robusto, así como interfaces de usuario intuitivas y funcionales.

Flujo de Trabajo

El flujo de trabajo para el “Usuario Arrendatario” implicará la gestión activa de propiedades, revisión de solicitudes de arrendamiento y seguimiento de pagos.

El “Usuario Arrendador” estará enfocado en búsqueda y arrendamiento de propiedades, así como gestión de pagos y calificación de experiencias.

Desarrollo de Plataforma

Deberá desarrollarse un backend exponiendo servicios en REST usando SPRING, además para la conexión con la base de datos se deberá usar JPA.

Repositorio y Fechas de Entrega

Los grupos deberán manejar un repositorio de control de versiones y cumplir con las fechas de entrega establecidas.

Fechas y Entregas

Entrega 1: JPA, Angular y Wikigroup

Fecha: 21 de septiembre 2023

  • JPA: Conexión a la base de datos.
  • Modelo entidad relación, en la wiki de su grupo ud deberá subir el modelo y el diccionario de datos.
  • Mapeo del modelo al proyecto JPA en Spring.
  • Pruebas unitarias del código, en la wiki ud deberá subir la documentación de los casos de pruebas en los que mínimo de información es: identificador de la prueba, título, descripción, diseñada por, ejecutada por, dependencias de las pruebas, resultados esperados, resultados reales y si cumplió o no cumplió. Notas deben ser por lo menos 5 pruebas por clase y se deben documentar si cumplió o no cumplió.
  • Debe realizar mínimo los servicios CRUD (Create, Retrieve, Update and Delete) de cada una de las clases que tiene; en la wiki debe tener los contratos funcionales de cada uno de los servicios y en debe traer en postman una consulta por cada servicio expuesto.

Entrega 2: Conexión JPA – Angular

Fecha: 19 de octubre 2023

Por ahora no es necesario realizar Autenticación, los datos se harán pensando que siempre lo realiza el mismo usuario.

  • Componentes: Deben mostrar componentes transversales, por lo menos 2.
  • Deben mostrar componentes principales de cada una de las páginas que tienen para cumplir con todas las historias de usuarios conectadas con el backend.
  • Routes: Se debe mostrar el mockup y mostrar cada una de las páginas que estén en el mismo.
  • Comunicación: Mostrar los modelos y servicios.
  • Comunicación backend: Verificación que desde la interfaz se realicen las historias de usuario.
